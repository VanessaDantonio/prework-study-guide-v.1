HTML

  • Hypertext Markup Language (HTML) is used to create the structure of a webpage.
  • The <!DOCTYPE html> declaration tells the browser what document type it should expect when it is opened.
  • The <html lang="en"> attribute tells the browser the language for the webpage.
  • The <head> element contains information about the webpage.
  • The <meta> elements contain information about the page that is used by the browser.
  • The <title> element defines the title for a webpage.
  • The <body> element represents the visible content shown to the user.
  • The <header> appears at the top of the page and usually contains a navigation menu and/or introductory content.
  • The <h1>, <h2>, <h3>, <h4>, <h5>, and <h6> elements represent the level of heading a given text block represents.
  • The <p> element represents a paragraph or block of text.
  • The <ul>, <ol>, and <li> elements represent unordered lists, ordered lists, and list items.
  • The <img> element contains information about images that are displayed on the webpage.
  • The ‘src’ attribute defines the location, or source, of the image file.
  • The ‘alt’ attribute contains a text string that describes the appearance and functionality of the image.
  • The <br> element creates a line of empty space, or a line break, between two blocks of content.
  • Speaking of the hyperlink, the <a> element (which stands for anchor) creates links to the same webpage or other webpages. The ‘href’ attribute points to the URL for the link.
  • The <footer> element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only. i.e. <h1 style="color:red;"> Prework Study Guide</h1>
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code. i.e.
    <head>
       <style>
          h1 {
             color: red;
          }
       </style>
    </head>
  • External style sheet allows developers to keep all their CSS rules in a separate file. When using an external style sheet, we link the file using a <link> element placed in the HTML's <head>. i.e.
    <head>
       <link rel="stylesheet" href="./assets/style.css"/>
    </head>
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property, separated with a colon (:).
  • Each ruleset must be wrapped in curly braces ({}). Within each ruleset, you must use a semicolon (;) to separate each declaration from the next one.
  • You can also select multiple elements and apply a single ruleset to all of them. Separate multiple selectors by commas (,).
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git config --global user.name "Your name": sets a Git username.
  • git config --global user.name: confirms that you have set the Git username correctly.
  • git config --global user.email "Your email": sets your email address.
  • git config --global user.email: confirms that you have set your email address correctly.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds all modifications in the current working branch to the staging area.
  • git commit -m "Descriptive message": commits your staged content as a new commit snapshot.
  • git pull origin main: pulls down any changes from the base branch.
  • git push origin branch-name: pushes your changes to the remote branch.

JavaScript

  • JavaScript is a programming language that allows users to interact with websites.
  • A variable is a named container that allows us to store data in our code. i.e. var topics = "HTML, CSS, Git, JavaScript";
  • Control flow is the order in which a computer executes code in a script.
  • console.log method outputs a message to the web console by adding an argument. The message may be a single string (with optional substitution values), or it may be any one or more JavaScript objects. i.e. console.log(topics);
  • To link your JavaScript file to the HTML file: add the code under the <footer> element and above the closing element for <body>. i.e.
          <footer>
             <p>I can code!</p>
          </footer>
          <script src="./assets/script.js"></script>
       </body>
    </html>
  • An array is a single variable that is used to hold a group of related data. i.e. var shapes = ["triangle", "square", "pentagon", "circle"];
  • Array index is used to access an individual data item inside the array. i.e. console.log(shapes[0]);
  • We can find the length of an array by using a built-in property of an array, appropriately named length.
  • The if statement executes a block of code if a specified condition is true. If the condition is false, another block of code can be executed. i.e.
    if (0) {
       console.log("I'm 0!")
    } else if (null) {
       console.log("I'm null!")
    } else if (true) {
       console.log("I'm true!")
    }
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. Developers must use another statement, such as the conditional if statement, to interrupt the control flow by executing a block of code a specific number of times. i.e.
    for(var x = 0; x < shapes.length; x++) {
       console.log(shapes[x]);
    }
  • A for loop contains the following three important statements:
    • The first statement determines the starting point for our loop. i.e. var x = 0;
    • The second statement is the condition. As long as the number in the variable x is less than the length of the array, the loop will keep running. i.e. x < shapes.length;
    • The final statement is what allows the array to iterate over each item. i.e. x++
  • The strict equality operator (= = =) checks to see if two values are equal, and returns a Boolean result true if the values are equal and false if the values are not equal.
  • We use a variable name plus “++” to iterate over each item, increasing the index number by 1 each time.
  • A function is a set of instructions that tells the computer how to perform a certain task. Functions do not automatically execute when the JavaScript file is run. Instead, you must call a function by name in order to execute it. i.e.
    function listTopics() {
       for (var x = 0; x < topics.length; x++) {
          console.log(topics[x]);
       }
    }
  • To call a function, we use the name of the function in our JavaScript file where we want the code to execute. i.e.
    console.log('Here are the topics we learned through Prework:');
    listTopics();
  • Math.floor() and Math.random() are both properties of JavaScript's built-in Math object that can be used together to generate a random number. i.e. var randomTopic = topics[Math.floor(Math.random() * topics.length)];
